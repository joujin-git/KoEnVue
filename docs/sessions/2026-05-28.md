- **cursor 잡티**: 사용자 한 줄 보고로 진단. 핵심은 `2528ce2` 에서 메인 `LayeredOverlayBase.ApplyPremultipliedAlpha` 의 "GDI AA 엣지 보존" 가드 (`a==0 && RGB!=0 → a=255`) 를 cursor 엔진에 의미 검토 없이 복사한 것. `ShadeDib` 의 sub-sample 가장자리 1개만 들어오는 픽셀이 `Math.Round(avgAlpha * 255) == 0` 으로 떨어지면 RGB 만 남은 부산 픽셀이 되고, 그 가드가 fully-opaque 점으로 변환 → cursor 가드 의미 정정 (RGB 도 0 으로 clear) + ShadeDib threshold 이중 방어. 학습: dev-notes 의 P4 예외 정당화 (메인/cursor 엔진 분리) 에서 *동일 함수명 가드의 의미는 별도 검토 필요* — cursor-indicator.md 의 "잠재 버그 fix" 절에 영구 기록
