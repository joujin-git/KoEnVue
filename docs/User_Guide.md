# KoEnVue 사용 설명서

Windows 한/영 입력 상태를 화면에 띄워주는 플로팅 인디케이터입니다.

- **한** — 한글 입력 모드
- **En** — 영문 입력 모드
- **EN** — 한국어 외 다른 IME (일본어 / 중국어 등)

CAPS LOCK이 켜져 있으면 라벨 **좌우에 세로 막대**가 함께 나타나, 한/영 상태와 CAPS LOCK 상태를 동시에 확인할 수 있습니다 (꺼지면 막대도 자동으로 사라집니다).

---

## 실행 방법

1. `KoEnVue.exe`를 원하는 폴더에 두고 더블클릭하면 끝납니다.
2. 시스템 트레이(작업 표시줄 우측)에 아이콘이 나타나고, 화면에 작은 인디케이터가 표시됩니다.
3. 첫 실행 시 `config.json`이 같은 폴더에 자동으로 만들어집니다.

> 관리자 권한이 필요합니다 (UAC 동의 창이 한 번 뜹니다).
> .NET 런타임 설치는 필요 없습니다 — 단일 실행파일입니다.

---

## 인디케이터 사용법

| 동작 | 설명 |
|------|------|
| **드래그** | 인디케이터를 마우스로 끌어 원하는 위치에 배치 |
| **앱별 위치 기억** | 앱마다 마지막에 둔 위치를 자동으로 기억 |
| **자석 스냅** | 드래그 중 다른 창 가장자리에 자석처럼 달라붙음 |
| **Shift + 드래그** | Shift를 누른 채 드래그하면 가로/세로 한 방향으로만 이동 |
| **드래그 활성 키 (선택)** | 기본 상태에선 인디 위 좌클릭이 전부 드래그를 개시. 트레이 "드래그 활성 키"에서 Ctrl/Alt/Ctrl+Alt 중 하나를 선택하면 해당 키를 누른 상태에서만 드래그 개시. 실수로 드래그되는 걸 막고 싶을 때 유용 |

> 멀티 모니터 / 고해상도(DPI) / 화면 회전 모두 자동으로 대응합니다.

## 트레이 아이콘 조작

| 조작 | 동작 |
|------|------|
| **좌클릭** | 기본은 **인디케이터 보임/숨김 토글**. 숨김 상태는 `config.json` 에 영구 저장돼 앱 재기동 후에도 유지됩니다. 숨겨진 상태에서는 트레이 아이콘 위에 **굵은 취소선 1줄**이 가로로 겹쳐 보여 "사용자가 명시적으로 숨겼음" 을 표시합니다. 다시 좌클릭하면 인디가 즉시 현재 앱에 다시 뜹니다. **상세 설정 → 트레이 → "좌클릭 동작"** 에서 `설정 파일 열기` (메모장으로 `config.json` 열기) 또는 `동작 없음` 으로 바꿀 수 있으며, 그 경우 숨김 해제는 아래 우클릭 메뉴의 **"인디케이터 숨김"** 항목으로 합니다 |
| **우클릭** | 트레이 메뉴 열기 (아래 참고) |

## 트레이 메뉴

트레이 아이콘을 **우클릭**하면 메뉴가 열립니다.

```
새 버전 있음 (v0.8.9.0) — 다운로드     ← 새 버전 감지 시에만 표시
───
투명도 ▸          진하게 / 보통 / 연하게
크기 ▸            1배 / 2배 / 3배 / 4배 / 5배 / 직접 지정...
☑ 창에 자석처럼 붙이기
☑ 애니메이션 사용
☑ 변경 시 강조
───
☑ 시작 프로그램 등록
───
기본 위치 ▸       현재 위치로 설정 / 초기화
위치 모드 ▸       ○ 고정 위치 / ● 창 기준
드래그 활성 키 ▸  ● 없음 / ○ Ctrl / ○ Alt / ○ Ctrl + Alt
위치 기록 정리...
───
☐ 인디케이터 숨김
───
상세 설정...
───
종료
```

| 항목 | 설명 |
|------|------|
| **새 버전 있음** | 앱 실행 시 GitHub 에 새 릴리스가 있으면 메뉴 최상단에 항목이 생깁니다. 클릭하면 기본 브라우저로 다운로드 페이지가 열립니다. (자동 업데이트는 아니며, 새 exe 를 직접 내려받아 교체해야 합니다.) 끄려면 **상세 설정 → 시스템 → "부팅 시 업데이트 확인"** 체크 해제 (또는 `config.json` 에서 `update_check_enabled: false`) |
| **투명도** | 인디케이터 배경 진하기 3단계 빠른 설정 |
| **크기** | 1~5배 정수 프리셋 + 소수점(예: 2.3배) 직접 입력 |
| **창에 자석처럼 붙이기** | 드래그 중 다른 창 엣지 스냅 켜기/끄기 |
| **시작 프로그램 등록** | Windows 로그온 시 자동 시작 (작업 스케줄러 등록) |
| **기본 위치** | 새 앱에서 인디케이터가 처음 뜰 위치 지정 (현재 위치 모드에 따라 고정/창 기준으로 저장) |
| **위치 모드** | **고정 위치**: 화면 절대 좌표에 고정. **창 기준**: 활성 창의 모서리 기준 상대 위치 — 같은 앱의 창을 여러 개 열어도 각 창에 맞춰 배치 |
| **드래그 활성 키** | 드래그 개시 게이트. **없음** (기본): 인디 위 좌클릭이 항상 드래그를 개시. **Ctrl / Alt / Ctrl + Alt**: 해당 키를 누른 상태에서만 드래그 개시. 키를 안 누른 상태의 클릭은 오버레이가 소비만 하고 반응 없음 (아래 창으로 전달되지 않음 — 크로스 프로세스 투과는 미지원). 실수로 인디가 끌리는 것을 방지하는 용도 |
| **위치 기록 정리...** | 앱별로 저장된 인디케이터 위치를 골라서 삭제 (실행 중인 앱은 "(실행 중)" 표시). 현재 위치 모드와 무관하게 고정/창 기준 양쪽 기록을 모두 표시하며, 삭제 시 양쪽에서 동시에 제거 |
| **인디케이터 숨김** | 좌클릭 토글과 동일한 `user_hidden` 체크 토글. 좌클릭 동작을 `설정 파일 열기` / `동작 없음` 으로 바꾼 환경에서도 항상 숨김 해제가 가능하도록 제공되는 안전망. 체크 시 인디가 즉시 숨겨지며(트레이 아이콘에 굵은 취소선), 해제하면 현재 앱에 바로 다시 뜹니다 |
| **상세 설정...** | 모든 설정을 한 화면에서 편집 |

## 자주 묻는 질문

**Q. 인디케이터가 어디로 갔는지 안 보여요.**
A. 트레이 아이콘 우클릭 → **기본 위치** → **초기화**를 누르면 기본 위치로 돌아옵니다.

**Q. 특정 앱에서만 인디케이터가 안 떠요.**
A. 전체화면(게임 등)이거나 포커스가 없는 창에서는 자동으로 숨겨집니다. **상세 설정**의 `hide_in_fullscreen`, `hide_when_no_focus`에서 끌 수 있습니다.

**Q. 색상이나 글꼴을 바꾸고 싶어요.**
A. 트레이 → **상세 설정**에서 6개 테마(`custom` / `minimal` / `vivid` / `pastel` / `dark` / `system`) 중 선택하거나 색상 코드(예: `#16A34A`)를 직접 입력할 수 있습니다. 기본값은 `custom` 이며, 테마를 `custom` 이 아닌 값으로 바꾸면 프리셋 색이 자동 적용됩니다. 프리셋을 쓰다가 `custom` 으로 돌아오면 이전에 설정했던 커스텀 색상이 자동으로 복원됩니다.

**Q. 설정을 직접 편집하고 싶어요.**
A. exe와 같은 폴더의 `config.json`을 텍스트 에디터로 열어 수정 후 저장하면 즉시 반영됩니다(앱 재시작 불필요).

일부 고급 옵션은 **상세 설정 대화상자에 나오지 않고 `config.json` 직접 편집으로만 조작**할 수 있습니다.

| 키 | 설명 | 예시 |
|----|------|------|
| `tray_enabled` | 트레이 아이콘 자체 켜고 끔. **OFF 로 두면 우클릭 메뉴가 사라지므로 이후 설정 변경은 `config.json` 재편집 또는 앱 재시작으로만 가능** | `false` |
| `user_hidden` | 사용자가 명시 숨김한 상태. 보통은 트레이 좌클릭(좌클릭 동작이 `toggle` 일 때) 또는 우클릭 메뉴 "인디케이터 숨김" 으로 토글하며, 파일을 직접 편집해서 `true` → `false` 로 바꿔도 해제 (hot reload + 다음 IME/포커스 이벤트 시 복원) | `false` |
| `system_hide_classes_user` | 기본 숨김 목록(바탕화면, 트레이 등) 외에 인디케이터를 숨기고 싶은 **윈도우 클래스명** 추가 목록 | `["MyAppWindow"]` |
| `system_hide_processes_user` | 같은 용도의 **프로세스명** 추가 목록 (확장자 제외, 대소문자 무관) | `["MyTool"]` |
| `app_filter_mode` + `app_filter_list` | 특정 앱에서만 인디케이터 표시/숨김. `"blacklist"` 는 목록 앱에서 숨김, `"whitelist"` 는 목록 앱에서만 표시 | `"blacklist"` + `["notepad"]` |
| `app_profiles` + `app_profile_match` | 특정 앱에서만 일부 설정(색·크기·투명도 등) 오버라이드. 아래 "앱별 프로필 예시" 참고 | (아래 예시) |
| `advanced.overlay_class_name` | 오버레이 윈도우 클래스명 (내부 디버깅용). 변경 권장하지 않음 | `"KoEnVueOverlay"` |

**앱별 프로필 예시**

`app_profiles` 는 명시한 앱에서만 일부 설정을 오버라이드하고 나머지는 전역값을 상속합니다. `app_profile_match` 로 키 형식을 선택합니다 — `process` (프로세스명 소문자, 기본) / `class` (윈도우 클래스명) / `title` (정규식, 100 ms ReDoS 방어).

```json
{
  "app_profile_match": "process",
  "app_profiles": {
    "notepad.exe": { "hangul_bg": "#DC2626", "english_bg": "#1D4ED8" },
    "chrome.exe":  { "indicator_scale": 3.0 },
    "devenv.exe":  { "enabled": false }
  }
}
```

notepad 는 한/영 배경색만 바꾸고, chrome 은 크기만 3 배로 키우고, devenv 는 `"enabled": false` 로 해당 앱에서만 프로필을 꺼서 전역 설정으로 동작합니다.

**Q. 완전히 삭제하려면?**
A. 트레이 → **시작 프로그램 등록** 해제 → **종료** → exe가 들어 있는 폴더 통째로 삭제. 설정과 로그도 같은 폴더에 있으므로 다른 곳에 흔적이 남지 않습니다.

## 파일 구성

| 파일 | 설명 |
|------|------|
| `KoEnVue.exe` | 본체 실행파일 (~4.7 MB) |
| `config.json` | 설정 파일 (자동 생성) |
| `koenvue.log` | 동작 로그 (문제 진단용, 자동 생성) |

## 시스템 요구사항

- Windows 10 / 11 (x64)
- 관리자 권한 (UAC 승인)
- 한국어 IME (Microsoft 한국어 IME 권장)
