# KoEnVue 사용 설명서

Windows 한/영 입력 상태를 화면에 띄워주는 플로팅 인디케이터입니다.

- **한** — 한글 입력 모드
- **En** — 영문 입력 모드
- **EN** — 한국어 외 다른 IME (일본어 / 중국어 등)

CAPS LOCK이 켜져 있으면 라벨 **좌우에 세로 막대**가 함께 나타나, 한/영 상태와 CAPS LOCK 상태를 동시에 확인할 수 있습니다 (꺼지면 막대도 자동으로 사라집니다).

---

## 실행 방법

1. `KoEnVue.exe`를 원하는 폴더에 두고 더블클릭하면 끝납니다.
2. 시스템 트레이(작업 표시줄 우측)에 아이콘이 나타나고, 화면에 작은 인디케이터가 표시됩니다.
3. 첫 실행 시 `config.json`이 같은 폴더에 자동으로 만들어집니다.

> v0.9.3.0 부터 관리자 권한이 필요 없습니다 — UAC 동의 창 없이 바로 실행됩니다.
> `Program Files` 처럼 사용자 쓰기 불가 위치에 두면 `config.json` 과 `koenvue.log` 가 `%LOCALAPPDATA%\KoEnVue\` 로 자동 이동합니다.
> .NET 런타임 설치는 필요 없습니다 — 단일 실행파일입니다.
> v0.9.4.0 부터 관리자 권한 cmd / 관리자 Windows Terminal 의 한/영 상태를 보려면 트레이 메뉴 **"관리자 권한으로 실행"** 을 켜세요 (선택, 기본 OFF — 자세한 내용은 아래 트레이 메뉴 절).

## 인디케이터 사용법

| 동작 | 설명 |
|------|------|
| **좌클릭 (짧은 클릭)** | 인디를 **일시 숨김**. 그 자리 마우스 클릭이 아래 창으로 통과합니다. 다른 창으로 포커스를 바꾸거나 한/영을 바꾸면 다시 표시. 트레이「플로팅 배지 숨김」(`user_hidden`)과는 별개 — 재시작 후에도 유지되지 않음 |
| **드래그** | 인디를 마우스로 끌어 원하는 위치에 배치 (시스템 드래그 임계를 넘기면 이동으로 인식) |
| **앱별 위치 기억** | 앱마다 마지막에 둔 위치를 자동으로 기억 |
| **자석 스냅** | 드래그 중 다른 창 가장자리에 자석처럼 달라붙음 |
| **Shift + 드래그** | Shift를 누른 채 드래그하면 가로/세로 한 방향으로만 이동 |
| **드래그 활성 키 (선택)** | 드래그 **승격** 게이트. 기본(없음)은 임계 초과 시 항상 드래그. 트레이에서 Ctrl/Alt/Ctrl+Alt 를 고르면 해당 키를 누른 채 끌 때만 드래그. 짧은 클릭은 항상 일시 숨김 |

> 멀티 모니터 / 고해상도(DPI) / 화면 회전 모두 자동으로 대응합니다.

## 트레이 아이콘 조작

| 조작 | 동작 |
|------|------|
| **좌클릭** | 기본은 **인디케이터 보임/숨김 토글**. 숨김 상태는 `config.json` 에 영구 저장돼 앱 재기동 후에도 유지됩니다. 숨겨진 상태에서는 트레이 아이콘 위에 **굵은 취소선 1줄**이 가로로 겹쳐 보여 "사용자가 명시적으로 숨겼음" 을 표시합니다. 다시 좌클릭하면 인디가 즉시 현재 앱에 다시 뜹니다. **상세 설정 → 트레이 → "좌클릭 동작"** 에서 `설정 파일 열기` (메모장으로 `config.json` 열기) 또는 `동작 없음` 으로 바꿀 수 있으며, 그 경우 숨김 해제는 아래 우클릭 메뉴의 **"플로팅 배지 숨김"** 항목으로 합니다 |
| **우클릭** | 트레이 메뉴 열기 (아래 참고) |

## 커서 헤일로

마우스 커서 주변에 **동심원 3개 + 흰색 헤일로** 를 띄워 한/영 상태를 커서 옆에서 확인할 수 있는 보조 인디입니다 (메인 라벨 인디와 동시 사용 가능). 기본 ON + 항상 표시 모드 — 첫 부팅부터 마우스 커서를 따라다니며 표시됩니다. 끄려면 트레이 메뉴 **"커서 헤일로 숨김"** 체크박스를 켜면 됩니다.

- **색상** — 안쪽 두 동심원은 현재 IME 색상 (한글 / 영문 / 비한국어 중 하나). 가장 바깥 원은 **CAPS LOCK 이 켜졌을 때만** 보이며, 색상은 "영문 IME → 한글 색상 / 한글·비한글 IME → 영문 색상" 으로 대조됩니다. CAPS 켠 채 한/영 토글 → 색상 교차로 한 눈에 식별
- **클릭 통과** — 커서 헤일로 위로 마우스 클릭/드래그가 항상 아래 창으로 통과합니다 (선택/스크롤 차단 없음). 플로팅 배지와 달리 드래그 이동 / 위치 저장 기능은 없음
- **표시 모드** — 두 가지 (`config.json` 의 `cursor_always_show` 키):
  - **항상 표시 (기본)** — 마우스를 따라 계속 위치 추종. 한/영 상태를 시야에서 놓치지 않게 함. 단, 작업 표시줄·시작/검색·클래식 컨텍스트 메뉴 위에서는 플로팅 배지·커서 헤일로가 함께 숨겨질 수 있음
  - **정지 시만 표시** — 마우스 정지 후 100ms 경과하면 인디 표시. 다시 움직이면 즉시 숨김. 작업 중에는 안 보이고 잠시 멈출 때만 확인 가능
- **전환 강조 (기본 ON)** — 한/영을 전환하면 동심원이 잠깐 커졌다 원래 크기로 돌아오는 강조 효과로 변경을 알려줍니다 (플로팅 배지의 전환 강조와 같은 동작). 끄려면 트레이 메뉴 **"커서 헤일로 변경 강조"** 체크를 해제하세요. 확대 배율·지속 시간은 **상세 설정 → "커서 헤일로 — 전환 효과"** 또는 `config.json` 의 `cursor_highlight_scale` / `cursor_highlight_duration_ms` 로 조정. 트레이 메뉴 **"애니메이션 사용"** 을 끄면 이 강조도 함께 멈춥니다 (메인·커서 모든 강조의 마스터 스위치). 또한 강조는 **한/영 상태가 실제로 바뀔 때만** 나타납니다 — 창만 바꾸는 경우(예: 한글 입력 중인 창에서 다른 한글 입력 창으로 전환)에는 상태 변화가 없으므로 강조가 뜨지 않습니다 (플로팅 배지와 같은 동작)
- **커서 헤일로 표시 (기본: 흐릿하게)** — 항상 표시 모드에서 동심원 선명도 3가지. 트레이 **"커서 헤일로 표시"** 또는 상세 설정 Combo · `cursor_display_mode`:
  - **흐릿하게** (`soft`, 기본) — 이동·정지 모두 가우시안 안개
  - **선명하게** (`sharp`) — 이동·정지 모두 또렷
  - **이동 중 흐릿하게** (`motion`) — 움직일 때만 안개, 멈추면 또렷(기존 PR-29/30)
  - 안개 농도/강도: 상세 설정「안개 농도」「안개 강도」또는 `cursor_motion_alpha` / `cursor_motion_softness` (선명하게 모드에서는 무시·값 보존)

세밀한 조정 (반지름 / 두께 / 헤일로 불투명도 / 전환 강조 / 커서 표시·안개 등) 은 트레이 메뉴 **"상세 설정"** 의 **"커서 헤일로 — 동심원"** / **"커서 헤일로 — 전환 효과"** 섹션 또는 `config.json` 의 `cursor_*` 키 직접 편집 ([config-reference.md → 커서 헤일로](config-reference.md#커서-헤일로-cursor-halo) 표 참고) 으로 가능합니다. 저장 즉시 반영됩니다.

## 자주 묻는 질문

**Q. 인디케이터가 어디로 갔는지 안 보여요.**
A. 트레이 아이콘 우클릭 → **기본 위치** → **초기화**를 누르면 기본 위치로 돌아옵니다.

**Q. 특정 앱에서만 인디케이터가 안 떠요.**
A. 전체화면(게임 등)이거나 포커스가 없는 창에서는 자동으로 숨겨집니다. **상세 설정**의 `hide_in_fullscreen`, `hide_when_no_focus`에서 끌 수 있습니다.

**Q. 색상이나 글꼴을 바꾸고 싶어요.**
A. 트레이 → **상세 설정**에서 6개 테마(`custom` / `minimal` / `vivid` / `pastel` / `dark` / `system`) 중 선택하거나 색상 코드(예: `#16A34A`)를 직접 입력할 수 있습니다. 기본값은 `custom` 이며, 테마를 `custom` 이 아닌 값으로 바꾸면 프리셋 색이 자동 적용됩니다. 프리셋을 쓰다가 `custom` 으로 돌아오면 이전에 설정했던 커스텀 색상이 자동으로 복원됩니다.

**Q. 설정을 직접 편집하고 싶어요.**
A. exe와 같은 폴더의 `config.json`을 텍스트 에디터로 열어 수정 후 저장하면 즉시 반영됩니다(앱 재시작 불필요).

일부 고급 옵션은 **상세 설정 대화상자에 나오지 않고 `config.json` 직접 편집으로만 조작**할 수 있습니다.

| 키 | 설명 | 예시 |
|----|------|------|
| `tray_enabled` | 트레이 아이콘 자체 켜고 끔. **OFF 로 두면 우클릭 메뉴가 사라지므로 이후 설정 변경은 `config.json` 재편집 또는 앱 재시작으로만 가능** | `false` |
| `user_hidden` | 사용자가 명시 숨김한 상태. 보통은 트레이 좌클릭(좌클릭 동작이 `toggle` 일 때) 또는 우클릭 메뉴 "플로팅 배지 숨김" 으로 토글하며, 파일을 직접 편집해서 `true` → `false` 로 바꿔도 해제 (hot reload + 다음 IME/포커스 이벤트 시 복원) | `false` |
| `system_hide_classes_user` | 기본 숨김 목록(바탕화면, 트레이 등) 외에 인디케이터를 숨기고 싶은 **윈도우 클래스명** 추가 목록 | `["MyAppWindow"]` |
| `system_hide_processes_user` | 같은 용도의 **프로세스명** 추가 목록 (확장자 제외, 대소문자 무관) | `["MyTool"]` |
| `app_filter_mode` + `app_filter_list` | 특정 앱에서만 인디케이터 표시/숨김. `"blacklist"` 는 목록 앱에서 숨김, `"whitelist"` 는 목록 앱에서만 표시 | `"blacklist"` + `["notepad"]` |
| `app_profiles` + `app_profile_match` | 특정 앱에서만 일부 설정(색·크기·투명도 등) 오버라이드. 아래 "앱별 프로필 예시" 참고 | (아래 예시) |
| `advanced.overlay_class_name` | 오버레이 윈도우 클래스명 (내부 디버깅용). 변경 권장하지 않음 | `"KoEnVueOverlay"` |

**앱별 프로필 예시**

`app_profiles` 는 명시한 앱에서만 일부 설정을 오버라이드하고 나머지는 전역값을 상속합니다. `app_profile_match` 로 키 형식을 선택합니다 — `process` (프로세스명 소문자, 기본) / `class` (윈도우 클래스명) / `title` (정규식, 100 ms ReDoS 방어).

```json
{
  "app_profile_match": "process",
  "app_profiles": {
    "notepad.exe": { "hangul_bg": "#DC2626", "english_bg": "#1D4ED8" },
    "chrome.exe":  { "indicator_scale": 3.0 },
    "devenv.exe":  { "enabled": false }
  }
}
```

notepad 는 한/영 배경색만 바꾸고, chrome 은 크기만 3 배로 키우고, devenv 는 `"enabled": false` 로 해당 앱에서만 프로필을 꺼서 전역 설정으로 동작합니다.

**Q. 완전히 삭제하려면?**
A. 트레이 → **시작 프로그램 등록** 해제 → **종료** → exe가 들어 있는 폴더 통째로 삭제. 설정과 로그도 같은 폴더에 있으므로 다른 곳에 흔적이 남지 않습니다.
